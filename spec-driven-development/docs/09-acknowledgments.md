# Acknowledgments

This project stands on the shoulders of giants. We're grateful for the following projects and communities.

## 🌟 Primary Inspiration

### **OpenSpec** - Fission-AI
**Repository:** [github.com/Fission-AI/OpenSpec](https://github.com/Fission-AI/OpenSpec/)

OpenSpec is a specification framework that makes executable documentation a reality. This entire project is built around demonstrating how to use OpenSpec effectively for Spec Driven Development.

**Why we're grateful:**
- ✅ Clear, human-readable specification format
- ✅ Bridge between business requirements and code
- ✅ Living documentation that stays synchronized
- ✅ Enables test generation from specifications
- ✅ Supports the Given-When-Then (Gherkin) pattern

**If you find this educational project helpful, please visit the [OpenSpec repository](https://github.com/Fission-AI/OpenSpec/) and consider:**
- ⭐ Starring the project
- 🔖 Following for updates
- 💬 Contributing if you have improvements
- 📖 Using it in your own projects

OpenSpec is the foundation that makes this learning project possible.

## 🧪 Testing Framework

### **pytest**
**Repository:** [github.com/pytest-dev/pytest](https://github.com/pytest-dev/pytest/)

The de facto standard for Python testing.

**Key features we use:**
- Simple, powerful assertion syntax
- Fixture support for test organization
- Plugin architecture for extensibility
- Excellent error messages
- Widely adopted across the Python ecosystem

### **pytest-bdd**
**Repository:** [github.com/pytest-dev/pytest-bdd](https://github.com/pytest-dev/pytest-bdd/)

BDD (Behavior Driven Development) support for pytest using Gherkin language.

**Key features we use:**
- Given-When-Then scenario support
- Automatic test generation from specifications
- Clear, readable test organization
- Integration with pytest ecosystem

Both tools are essential to this project's test-driven approach.

## 📦 Data Validation

### **Pydantic**
**Repository:** [github.com/pydantic/pydantic](https://github.com/pydantic/pydantic/)

Data validation and parsing using Python type hints.

**Key features we use:**
- Type checking at runtime
- Automatic validation of model fields
- Custom validators for business rules
- Serialization to/from JSON
- Clear error messages for validation failures

Pydantic ensures our data models follow the specification constraints.

## 📚 Methodology Influences

This project is influenced by:

### **Test-Driven Development (TDD)**
- Kent Beck's seminal work on writing tests first
- The Red-Green-Refactor cycle
- Building confidence through tests

### **Behavior-Driven Development (BDD)**
- Dan North's Given-When-Then pattern
- Cucumber framework approach
- Focusing on business value and behavior

### **Specification-Driven Development (SDD)**
- Marrying specifications and tests
- Living documentation
- Single source of truth
